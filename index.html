Сетевая модель OSI

Модель OSI (Open Systems Interconnection) — концептуальная модель, характеризующая и стандартизирующая коммуникационные функции телекоммуникационной системы без зависимости от её внутренней структуры и технологий.

7 уровней модели OSI

Прикладной (Application)
HTTP, HTTPS, SMTP, DNS, FTP. Взаимодействие с приложениями.
Представления (Presentation)
MIME, XDR, ASN.1. Кодирование, шифрование, сжатие данных.
Сеансовый (Session)
NetBIOS, RPC, SIP. Управление сеансами связи.
Транспортный (Transport)
TCP, UDP. Надёжная передача данных, управление потоком.
Сетевой (Network)
IPv4, IPv6, ICMP, ARP. Маршрутизация и логическая адресация.
Канальный (Data Link)
Ethernet, PPP, HDLC. Обнаружение ошибок, управление доступом.
Физический (Physical)
IEEE 802.15 (Bluetooth), RS-232, Wi-Fi. Передача битов по среде.

Протокол HTTP

HTTP (Hypertext Transfer Protocol) — протокол передачи гипертекстовых документов по схеме «запрос-ответ». Используется для обмена данными в веб.

Защищённый HTTPS

HTTPS (HyperText Transfer Protocol Secure) — расширение HTTP с поддержкой шифрования через SSL/TLS для защиты передаваемых данных.

Параметры DV SSL (Подтверждённый домен) OV SSL (Подтверждённая организация) EV SSL (Расширенное подтверждение)
Протоколы HTTPS HTTPS HTTPS + отображение организации в браузере
Проверка Только домен Домен + организация Полная юридическая проверка
Поддомены Разрешены любые (Wildcard) Только включённые в сертификат Только включённые в сертификат
Использование Блоги, личные сайты Интернет-магазины, компании Банки, платежные системы

HTTP/3 и протокол QUIC

HTTP/3 использует транспортный протокол QUIC поверх UDP для уменьшения задержек и улучшения производительности.

HTTP/1.0 1996 TCP Заголовки, методы GET/POST/HEAD
HTTP/1.1 1999 TCP Постоянные соединения, кэширование
HTTP/2 2015 TCP Мультиплексирование, Server Push
